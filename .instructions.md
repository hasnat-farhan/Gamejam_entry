# Project Instructions for Godot 4.7

This repository is a Godot 4.7 stable project using text scene format (`.tscn`). Assistants should follow these rules when editing or generating code and assets.

## Godot 4 GDScript conventions
- Use Godot 4 syntax only.
- Prefer `@export` annotations instead of the removed `export` keyword.
- Use `create_tween()` instead of creating `Tween` directly.
- Keep GDScript parse compatibility with Godot 4.7 stable.

## .tscn and asset handling
- Maintain valid `.tscn` subresource order for Godot 4: define `AtlasTexture` subresources before any `SpriteFrames` or other resources that reference them.
- Keep `res://` root-relative paths consistent across scenes and scripts.
- When fixing missing references, search the workspace for existing asset files and update paths to the correct `res://` location rather than adding duplicate assets.
- Avoid arbitrary legacy `godot-dungeon-master` or addon modifications unless the user specifically requests them.

## Project-specific priorities
- Fix broken generated asset scenes from `adeeb assets` imports first.
- Resolve missing-file parse errors by repairing paths and Godot 4 compatibility issues.
- Use placeholder files only when required to satisfy missing `res://` references and when no matching asset exists.

## When unsure
- Ask the user whether changes should be limited to the active game content or if legacy/example files may also be updated.
- Prefer small, targeted fixes rather than broad changes across the whole workspace.
